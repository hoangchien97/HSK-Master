# Sprint 7 — Admin: Users + Analytics

## Goal
SYSTEM_ADMIN quản lý user & xem analytics cơ bản (học viên, attendance, submissions).

## Scope
### 1) User management (SYSTEM_ADMIN)
- List users + filter role/status
- Promote/demote role
- Lock/unlock user
- View user detail (classes, activity)

### 2) Analytics
- KPIs: active students (30d), attendance rate, on-time submission rate
- Charts monthly (basic)
- Export CSV (optional)

## Acceptance Criteria
- [ ] Only SYSTEM_ADMIN can access
- [ ] Role change affects routing immediately
- [ ] Analytics matches DB

## Tasks
- [ ] Admin pages UI (table + filters)
- [ ] Admin APIs
- [ ] Aggregation queries
